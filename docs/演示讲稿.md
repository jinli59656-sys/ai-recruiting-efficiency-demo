# 面试 Demo 演示讲稿

## 2-3 分钟演示顺序

### 1. 开场

这个项目是一个 AI 招聘提效 Demo，目标是解决 HR 在企业微信、招聘群和腾讯在线文档之间手工搬运招聘数据的问题。

核心流程是：

```text
企业微信群消息 -> AI 结构化抽取 -> 招聘数据入库 -> 腾讯文档镜像同步 -> 招聘看板更新
```

这个方案的重点不是做一个大而全的 HR 系统，而是用最小成本把招聘数据记录、同步和可视化这条链路自动化。

### 2. 登录并进入数据采集

打开系统后点击“演示模式登录”，进入“数据采集”页面。这里模拟的是企业微信群中的招聘协作消息。

### 3. 选择示例消息

选择示例消息：

```text
@HR 张三 Java后端 3年经验 本科，简历已收，约周三14:00初面，面试官王经理
```

这类消息在真实招聘群里非常常见，里面同时包含候选人、岗位、年限、学历、面试时间和面试官。

### 4. AI 解析并同步

点击“AI 解析并同步”。系统会自动抽取结构化字段：

- 候选人：张三
- 岗位：Java后端
- 学历：本科
- 年限：3年
- 阶段：初面
- 时间：周三14:00
- 面试官：王经理

如果配置了模型 Key，系统优先调用 Qwen 做结构化抽取；如果没有模型 Key 或模型异常，系统会使用规则兜底解析，并标记“需人工确认”。

### 5. 展示腾讯文档镜像表

解析完成后，下方“腾讯在线文档镜像表”会自动新增一行数据。Demo 阶段这里用数据库表模拟腾讯文档，正式落地时可以替换为腾讯文档 API。

这个设计的好处是：HR 不需要改变原有工作习惯，仍然在企业微信和腾讯文档里协作，但中间的重复录入被 AI 自动化替代。

### 6. 展示工作台

回到工作台，可以看到今日自动采集数、文档同步总数、需人工确认数和最近同步动态。管理者可以实时看到招聘进展，不再依赖 HR 手动维护统计表。

### 7. 总结

这个 Demo 展示的是一条低成本、可落地的招聘提效路径：

- 企业微信作为信息入口。
- AI 负责非结构化文本抽取。
- 腾讯文档作为协作载体。
- 看板负责实时统计和异常提醒。
- 人工确认机制控制 AI 错误风险。

## 面试官可能追问

**Q：是否真的接入了腾讯文档？**

Demo 阶段没有直接写真实腾讯文档，而是用数据库表模拟腾讯文档镜像，验证字段抽取、同步状态、人工确认和看板更新闭环。正式落地时只需要把同步适配器替换为腾讯文档 API。

**Q：模型抽错怎么办？**

系统有 `confidence` 和 `needs_review` 字段。低置信度或规则兜底解析的数据会标记为“需人工确认”，HR 确认后再进入后续流程。

**Q：没有模型 Key 是否还能跑？**

可以。系统会自动走规则兜底解析，Demo 不依赖模型服务可用性。

**Q：方案的最小成本体现在哪里？**

不采购新的 ATS，不替换现有企业微信和腾讯文档，只增加一个轻量 AI 同步服务，按量调用模型，并保留人工确认机制。
